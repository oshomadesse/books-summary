### 🗣️ 要約
- 📣 核心的メッセージ
```
経営理論は「正解」を教えるのではなく、複雑な現実を解くための検証済みの視点＝道具箱である。状況依存（コンティンジェンシー）を前提に、産業・資源・組織・ネットワーク・制度を横断して理論を組み合わせ、仮説検証を回し続けることが競争優位を生む。
```

- 🖊️ エグゼクティブ・サマリー
```
問い1: なぜ経営理論を学ぶべきか？
答え: 意思決定の質と再現可能性を上げ、不確実性下でも学習速度を高めるため。
Why: 人はバイアスに陥りやすく、理論は大規模な検証・反証で一般化された「うまくいく原理」を提供するから。
How: 状況診断→適用候補理論の仮説化→小さく実験→学習・更新のループを制度化する。

問い2: 何を学ぶべきか？
答え: 価値獲得のポジショニング、価値創造の資源・知識、持続化のケイパビリティ、変革の両利き・オープン、拡張のネットワーク・プラットフォーム、境界・ガバナンス・制度・行動の理論群。
Why: 価値の創造→獲得→維持→拡張→変革という経営の全課題を網羅する中核メカニズムだから。
How: 産業構造×RBV×DC×両利き×ネットワーク×TCE×制度×行動を「補完し合うモジュール」として理解し、ケースに応じて最小有効セットを設計する。

問い3: どう使うか？
答え: 一義のベストプラクティスを探すのではなく、理論を組み合わせた仮説を素早く検証・捨て替えする実験志向で運用する。
Why: 環境変化と経路依存性が高く、単独理論の過信は誤るため。
How: 5フォースで収益性仮説→RBVで独自資源の仮説→DCで再構成計画→両利きで探索と深化の配分→ネットワーク外部性・補完資産でスケール設計、の順で小さく回す。
```
