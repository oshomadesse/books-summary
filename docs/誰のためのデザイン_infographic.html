📘 誰のためのデザイン？

認知科学者が解く、人間中心設計の原則

🎯 核心メッセージ

❓ 問い

なぜ日常の製品やUIは使いにくいのか？

💡 答え

デザインが人の心的モデル・注意資源・学習特性と不整合で、発見可能性・理解可能性・誤り耐性が不足するから。

🔍 Why（なぜそうなのか）

人の行為は「目標→意図→行為仕様→実行→知覚→解釈→評価」の7段階をたどり、その間に実行の溝と評価の溝が生じる。可視性が低い・マッピングが悪い・シグニファイアが曖昧だと誤操作とフラストレーションを誘発する。

🛠️ How（そのためには）

アフォーダンスを明確化し、適切なシグニファイア・ナチュラルマッピング・即時のフィードバック・多層の制約（物理/意味/文化/論理）・エラーに強い仕組み（フォース機能/インターロック/ロックイン）を設け、観察→仮説→プロトタイピング→ユーザテスト→反復のHCDプロセスで磨き込む。

🎨 人間中心設計（HCD）

  • 観察とテストに基づく反復設計
  • 人の目標・文脈・制約に合わせる
  • 機能より人間に焦点を当てる
理由: 技術起点でなく人起点で設計することで、実際の使用文脈での成功率と満足度が向上する。

🤲 アフォーダンス

  • 環境や物が提供する可能な行為
  • ユーザが何ができるかを規定
  • 本来的な性質（押せる、引ける等）
理由: 物理的・デジタル的特性が行為の可能性を決めるため、設計の出発点となる。

🏷️ シグニファイア

  • どの行為がどこで可能かを示す手掛かり
  • ラベル、形、音、色などで伝達
  • アフォーダンスを可視化する
理由: アフォーダンスが存在しても知覚できなければ使えない。シグニファイアが発見可能性を実現する。

🗺️ ナチュラルマッピング

  • 制御と結果の対応が直感的
  • 空間や論理の類似で配置
  • 学習不要で操作できる
理由: 人の空間認知や因果推論と整合すると、記憶負荷なく即座に操作できる。

🔔 フィードバック

  • 操作の結果を即時・意味ある形で返す
  • 状態と因果を可視化
  • 評価の溝を埋める
理由: 行為の結果が見えないと、ユーザは次の行動を決められず不安とエラーが増える。

🚧 制約（物理/意味/文化/論理）

  • 不適切な行為を防ぐ仕組み
  • 選択肢を絞り、誤りを減らす
  • 学習負荷を軽減
理由: 誤った操作を物理的・論理的に不可能にすることで、安全性と効率が向上する。

🧠 概念モデル

  • ユーザが心中に持つシステムの動作モデル
  • 設計側のモデルと一致が重要
  • 因果関係の理解を支える
理由: ユーザの心的モデルと設計意図が一致すると、予測可能性が高まりエラーが減る。

🔍 発見しやすさ

  • 何ができるか・どう操作するかが見てわかる
  • シグニファイアとマッピングの総合
  • 学習時間を最小化
理由: 初見のユーザでも迷わず使えることが、普及と満足度の鍵となる。

🔄 行為の7段階

  • 目標→意図→行為仕様→実行→知覚→解釈→評価
  • 各段階で障害が起きる
  • 実行と評価の溝を特定
理由: 行為プロセスを分解することで、どこに設計介入が必要かを診断できる。

⚡ 実行の溝／評価の溝

  • 実行の溝: 目標から行為への隔たり
  • 評価の溝: 世界の状態を解釈する隔たり
  • よいデザインは両方を橋渡し
理由: 溝が広いほど学習コストとエラー率が上がるため、設計の最優先課題となる。

❌ スリップとミステイク

  • スリップ: 意図は正しいが実行で誤る
  • ミステイク: 意図・判断自体が誤る
  • それぞれ異なる対策が必要
理由: エラーの種類を区別することで、制約やフィードバックの設計を最適化できる。

🛡️ エラーに強い設計

  • 誤りを防止・軽減し、復帰を容易に
  • フォース機能、インターロック、ロックイン
  • 重大事故の確率・影響を低減
理由: 人は必ず誤るため、システム側で誤りを吸収する仕組みが安全性の要となる。

📊 概念間の関係性

アフォーダンス + シグニファイア → 発見しやすさ↑ → 実行の溝が狭まる
ナチュラルマッピング + 制約 → 誤操作↓ → 学習時間↓
フィードバックの即時性・意味性↑ → 評価の溝が狭まる → メンタルモデル整合
概念モデル（設計者=ユーザ）一致 → スリップ/ミステイク↓
知識（世界の中）を増やす → 記憶負荷↓ → エラー率↓
エラーに強い設計 → 重大事故の確率・影響↓
HCDの反復（観察→プロトタイプ→テスト） → 不適合の早期検出 → 品質向上・コスト低減

第1章: 日常に潜む使いにくさ

身近な失敗例から、可視性・マッピング・フィードバック欠如が問題の根と指摘。ユーザ非難ではなく設計の責任を強調。

理由: 使いにくさの責任を設計に置くことで、改善の起点を明確にし、ユーザ体験向上の動機づけを与える。

第2章: 行為の心理学

行為の7段階と実行/評価の溝を提示。よいデザインはこの二つの溝を橋渡しする。

理由: 認知プロセスを分解することで、設計介入のポイントを科学的に特定できる。

第3章: 頭の中と世界の中の知識

指示・制約・標準化など、世界側の支援で記憶負荷を減らし、操作を容易にする戦略を解説。

理由: 外部化・標準化により、ユーザの認知負荷を下げ、誤りと学習時間を削減できる。

第4章: 何をすべきかをわからせる仕組み

アフォーダンス、シグニファイア、制約、ナチュラルマッピングを体系化。発見可能性の設計法を示す。

理由: 発見可能性を高めることで、初見ユーザでも迷わず使える製品を実現できる。

第5章: 人は誤る

スリップ/ミステイクの分類と誘発要因を説明。エラーに強い設計（フォース機能、インターロック、ロックイン）を提案。

理由: エラーを前提とした設計により、安全性と信頼性を大幅に向上できる。

第6章: デザイン思考とプロセス

観察・仮説・プロトタイピング・ユーザテストの反復を中心とするHCDの具体プロセスを提示。

理由: 反復的な検証サイクルにより、不適合を早期発見し、品質とコストを最適化できる。

第7章: ビジネスと組織の現実

コスト・スケジュール・マーケティングなど組織制約下で原則を適用する方法と妥協の指針を論じる。

理由: 理想と現実のバランスを取る方法を示すことで、実務での実践可能性を高める。

第8章: 複雑化する世界のデザイン

ネットワーク化・自律化時代における可視性、責任、説明可能性の重要性を展望。

理由: 技術進化に伴う新たな課題に対し、人間中心設計の原則を拡張・適用する必要性を示す。

🚪 例1: ドアの押す/引くの混乱

背景: 多くのドアは押すべきか引くべきか見ただけではわからず、ユーザは試行錯誤を強いられる。

示唆: ハンドルの形状（平板=押す、取っ手=引く）というシグニファイアが不適切だと、アフォーダンスが伝わらない。

理由: この例は、シグニファイアとナチュラルマッピングの不一致が日常的なフラストレーションを生む典型を示す。

🔥 例2: コンロのつまみ配置

背景: 4つのコンロに対し、つまみが一列に並んでいると、どのつまみがどのコンロを制御するか直感的にわからない。

示唆: つまみをコンロの空間配置と対応させる（ナチュラルマッピング）ことで、誤操作と学習負荷が激減する。

理由: 空間的対応の欠如が誤操作を誘発し、安全性と効率を損なうことを具体的に示す。

💾 例3: 保存ボタンのフィードバック欠如

背景: ユーザが保存ボタンを押しても、視覚的・音声的フィードバックがないと、本当に保存されたか不安になり、何度も押してしまう。

示唆: 即時のフィードバック（チェックマーク、音、メッセージ）が評価の溝を埋め、安心感と効率を高める。

理由: フィードバックの有無が、ユーザの確信と次の行動決定に直結することを示す。
"使い手を責めるな。責めるべきは悪いデザインだ。"

重要性: 責任の所在を設計に置き、改善の起点を明確にする。

"よいデザインは使い方を説明不要にする。"

重要性: 発見可能性とナチュラルマッピングの目標状態を示す。

"人は必ず誤る。だからシステムを誤りに強くせよ。"

重要性: ヒューマンエラー前提の安全設計を促す。

"知識は頭の中だけでなく世界の中にも置ける。"

重要性: 外部化・標準化で記憶負荷を下げる方策を正当化する。

"観察し、つくり、試し、学び、またつくる。"

重要性: HCDの反復サイクルの核心を簡潔に示す。

🚪 アクション1: ドア1枚のシグニファイアを点検

自宅や職場のドアを1つ選び、押す/引くの手掛かり（ハンドル形状、ラベル、視覚的手掛かり）を観察し、改善案をメモする。

目的: シグニファイアとアフォーダンスの関係を体感的に理解する。
期待効果: 日常の使いにくさを設計の視点で分析する習慣が身につく。

📱 アクション2: よく使うアプリのフィードバック確認

スマホアプリで1つの操作（保存、送信、削除等）を行い、フィードバック（視覚、音、振動）の有無と質を評価する。

目的: フィードバックが評価の溝を埋める役割を実感する。
期待効果: 自分の設計にフィードバックを意識的に組み込めるようになる。

📝 アクション3: 自作タスクの7段階を紙に書く

今日行う1つのタスク（例: メール送信）を「目標→意図→行為仕様→実行→知覚→解釈→評価」の7段階に分解し、どこに障害があるか特定する。

目的: 行為の7段階モデルを実践的に理解する。
期待効果: 設計時に実行と評価の溝を意識し、介入ポイントを見つけられるようになる。
